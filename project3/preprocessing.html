Explain the ways to implement preprocessing

Preprocessing can be done through either usages of SASS or LESS. Both scripting languages serve as a shortcut to enable easy maintence of CSS. LESS is processed in Node.JS, while SASS can be preprocessed in Node.JS too; but with additional applications such as Codekit or Chocolatey. This difference is important, because Node.JS is seen as a balance between client-sided and server-sided scripts within the operating system. Node.JS focuses mainly on returning server content to the client. This makes things super simple for LESS. Considering there being a time where SASS used to be done through RUBY, shows how heavy on the server side the processing was. Server-sided components usually don't have to be written in Javascript, this is because the server supports other languages. Having one avenue to complete a task in CSS, makes LESS more appealing.
